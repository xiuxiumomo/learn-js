## 对象 key 数据的四个属性

**了解四个属性**

- 1.Configurable：表示是否可以通过 delete 删除并重新定义，默认情况下是 true

- 2.Enumberable: 表示该属性是否可以通过 for-in 打印出来

- 3.Writable: 表示是否可以重写该属性

- 4.value: 默认值 undefined

```
let person = {}

Object.defineProperty(person,'name',{
  configurable: true,
  enumerable: true,
  writable: true,
  value: undefined
})
## 不允许删除
Object.defineProperty(person,'name',{
  configurable: false,
  enumerable: true,
  writable: true,
  value: undefined
})
person.name = 'jack'
delete person.name
person.name //jack

## 不允许for-in

Object.defineProperty(person,'name',{
  configurable: true,
  enumerable: false,
  writable: true,
  value: undefined
})
person.age = 20

for(let k in person) {
  console.log(k)
}
// age


## 不允许重新复制
Object.defineProperty(person,'name',{
  configurable: true,
  enumerable: true,
  writable: false,
  value: 'jack'
})

person.name = 'pack'
person.name  // jack

## 指定默认值
Object.defineProperty(person,'name',{
  configurable: true,
  enumerable: true,
  writable: true,
  value: 'jack'
})

person.name //jack

```

**访问器属性**

- 1.configuable 表示是否可以通过 delete 删除属性
- 2.enumerable 表示是否可以通过 for in 打印该属性

- get 方法属性 默认值为 undefined
- set 方法属性 默认值为 undefined

```
let book = {
  year_: 2017,
  edition: 1
}
Object.defineProperty(book,'year',{
  get(){
    return this.year_
  },
  set(val){
    if(val>2017) {
      this.year_ = val;
      this.edition+=1
    }
  }
})
book.year = 2018;
console.log(book.edition)

```

**合并对象**

> JavaScript 开发者经常觉得“合并”（merge）两个对象很有用。更具体地说，就是把源对象所有的
> 本地属性一起复制到目标对象上。有时候这种操作也被称为“混入”（mixin），因为目标对象通过混入
> 源对象的属性得到了增强

```
dest = {}; 
src = { id: 'src' }; 
result = Object.assign(dest, src);

```

## 创建对象

**工厂模式**

```
function person(name,age) {
  let o = new Object();
  o.name = name;
  o.age = age;
  o.sayName = function() {
    return this.name
  }
  return o;
}

let p = person('jack',20)
console.log(p.name)
```

**构造函数**
构造函数的特点

- 1.没有显示的return方法
- 2.没有显示的创建新的对象
- 3.属性和方法直接给this

new关键字做了什么操作？

- 1.在内存中创建一个新的对象
- 2.新对象的prototype属性被赋值为构造函数的prototype属性
- 3.构造函数内部this被赋予为新的对象
- 4.执行构造函数的代码
- 5.如果构造函数非空对象，返回该对象，否在返回新创建的对象
```
function Person(name,age) {
  this.name = name;
  this.age = age;
  this.sayName = function() {
    return this.name
  }
}


let p = new Person('jack',20)

console.log(p.sayName())
```

**理解什么是原型？**

默认的，所有的function A(){} 都会获得原型属性prototype它是一个对象 A.prototype且它的第一个属性constructor 指向 A方法。
在Firefox,chrome,safaria还会暴露出一个 __proto__ 属性，该属性是一个完全隐藏的属性。

```
function Person() {

}

console.log(Person.prototype)

//结果
{
  constructor: f Person,
  __proto__: Object
}

console.log(Person.prototype.__proto__ === Object.prototype); // 继承object上的属性和方法
console.log(Person.prototype.__proto__.constructor === Object); // true 
console.log(Person.prototype.__proto__.__proto__ === null); // true
```

构造函数出来的原型关系

```
function Persion() {}

let p = new Person();

Person.prototype.constructor === Persion //true

p.__proto__ === Person.prototype //true

p.__proto__.constructor === Person //true
```

- hasOwnProperty方法检测某个属性来自自己还是来自原型

```
function Person(name, age) {
  this.name = name;
  this.age = age;
  this.sayName = function () {
    return this.name;
  };
}
Person.prototype.sex = 'man'
let p = new Person('jack',20)
console.log(p.hasOwnProperty('name')) //true
console.log(p.hasOwnProperty('sex')) //false
console.log('sex' in p) //true

```
